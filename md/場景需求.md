1. 目標

建立一套可在 Isaac Sim 穩定產生的資料集，用來訓練或補強：

草地分割
草地 vs 泥土 / 枯草 / 落葉 / 石頭 / 障礙物
固定車載相機視角
連續影片幀
可擴充到 temporal model 或後處理驗證
2. 類別定義

先不要分太細，第一版建議這樣。

二分類版
0: background
1: grass

適合先把草地區域穩定抓出來。

三到五類版

比較推薦這版，後續比較好用：

0: background
1: grass
2: soil
3: obstacle
4: unknown 或 curb/road

如果你未來要做導航 costmap，這版比純二分類更實用。

3. 場景配置

至少準備 4 種場景，不要只做一片漂亮草地。

場景 A：標準草地
大面積綠草
少量不規則草高
偶爾有裸土
場景 B：草土混合
草地內混裸土
有輪胎壓痕
有濕地或顏色較深區域
場景 C：枯草與落葉
黃綠混雜
局部落葉
草地顏色不均
場景 D：障礙物場景
石頭
樹幹
花圃邊界
路緣
短樁、箱子、假人、園藝設備

你的目標不是做漂亮 demo，是讓模型看過會搞混的東西。

4. 相機規格

這裡要盡量貼近你真機。

固定參數
相機高度：依你車體，例 0.35 ~ 0.7 m
俯角：例 20° ~ 45°
水平 FOV：依實機鏡頭
解析度：建議同訓練尺寸前的原始輸出一致，例如 1280x720 或 960x540
建議輸出

每個 sample 輸出：

RGB
semantic segmentation
depth
可選 normals
可選 instance segmentation

depth 雖然你現在未必拿來訓練，但後面做 projection 或 ROI 會很好用。

5. 時序資料規格

因為你有影片穩定性問題，資料最好不是只有單張。

第一版建議

每段 clip：

長度：30 ~ 100 幀
FPS：10 ~ 20
相機沿 mower 前進路徑移動
有直行、緩轉、轉彎、停頓
每段 clip 要保留
clip_id
frame_id
timestamp
camera pose
可選 robot pose

這樣你後面要做 temporal smoothing、warp、長序列模型都方便。

6. Domain Randomization

這是 Isaac Sim 最有價值的地方，重點是亂得有意義。

光照

每個場景要隨機：

太陽高度
太陽方向
光照強度
陰影長短
晴天 / 陰天
局部遮蔽陰影
地表材質
草色深淺
草長短
枯草比例
泥土顏色
濕地反光程度
落葉密度
障礙物
障礙物數量
位置
尺寸
材質
顏色
相機擾動
輕微 pitch / yaw / roll 抖動
motion blur
noise
曝光變化
白平衡漂移
gamma 變動

這一塊很重要，因為你真機影片誤判很多時候就來自相機特性。

7. 資料比例建議

不要平均亂配，要偏向 hard case。

建議比例
40% 正常草地
25% 草土混合
15% 枯草/落葉
10% 強陰影
10% 障礙物密集或視角困難

如果你目前真機最容易在陰影下誤判，那陰影資料比例就再往上拉。

8. 輸出格式

資料夾建議長這樣：

dataset/
  train/
    clip_0001/
      rgb/
      seg/
      depth/
      meta/
    clip_0002/
  val/
    clip_0101/
  test/
    clip_0201/
檔案命名
rgb_000001.png
seg_000001.png
depth_000001.exr
meta_000001.json
meta 內容建議
{
  "clip_id": "clip_0001",
  "frame_id": 1,
  "timestamp": 0.05,
  "camera_pose": {...},
  "lighting_profile": "low_sun_shadow",
  "terrain_profile": "grass_soil_mix",
  "weather_profile": "clear"
}

這樣後面比較好針對錯誤場景回查。

9. 標註規則

semantic label 一定要固定，不要一套場景一種名字。

例如統一：

grass
soil
obstacle
curb
road

不要有時候叫 grass_patch，有時候叫 lawn，有時候叫 green_area。
你匯出前就先把語意 mapping 統一。

10. 訓練用途分兩條
路線 A：直接混合訓練
真實資料為主
Isaac Sim 資料為輔
比例先抓 7:3 或 8:2
路線 B：先 synthetic pretrain，再 real finetune
先用 Isaac Sim 學基礎語意
再用真實草地資料收斂到真機分布

如果你真實標註量少，這條通常很有價值。

11. 驗證集怎麼留

val / test 不要只留一般場景，要故意留難的。

驗證至少要分這幾組
normal
shadow
grass_soil_mix
dry_grass
obstacle_dense
motion_blur

這樣你才知道模型到底是「平均不錯」還是「一遇到陰影就壞」。

12. 對你目前最有幫助的輸出

如果你現在優先是影片表現，我建議你在 Isaac Sim 至少輸出這三樣：

RGB
semantic mask
clip sequence metadata

重點不是只做單張圖，而是要做 連續 clip，這樣你可以直接拿來驗證：

前後幀連續性
temporal smoothing
probability map 穩定性
長序列模型可行性
13. 第一版最小可行規模

不要一開始就做很大，先做可用版本。

第一版建議
4 種場景
每種 20 段 clip
每段 50 幀
共約 4000 幀

如果要更穩：

4 種場景
每種 50 段 clip
每段 50 幀
共約 10000 幀

先做這種量級就夠你驗證方向了。

14. 最後提醒

Isaac Sim 最有價值的不是「完全取代真實資料」，而是：

幫你快速生 hard case
產生穩定一致的語意標註
補時序資料
讓你做相機與光照條件控制

對你這種任務，最合理的是：

真實資料打底 + Isaac Sim 補 hard case + 最後真機驗證
